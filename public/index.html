Privacy & cryptography

Privacy

  • Client-side only — the page never sends generated passwords anywhere. Open DevTools / Network and verify yourself.no XHR, no fetch, no beacon, no third-party scripts
  • No analytics — no Google Analytics, no Plausible, no fingerprinting, no cookies.
  • No logs — the nginx that serves this page logs HTTP requests (path, status, user agent), not anything you type into the form.
  • Source is public — verify the claim:
    github.com/AaronShemtov/pwd-frontend

Cryptography

  • CSPRNG — randomness is sourced from window.crypto.getRandomValues(), backed by the OS-level entropy pool (Linux /dev/urandom, Windows BCryptGenRandom).never Math.random — that's a PRNG, predictable after a few samples
  • Uniform distribution — index into the charset uses rejection sampling against uint32. No modulo bias.`random % N` is biased whenever N doesn't divide 2³²
  • Class guarantee — if a character class is selected but happened to be absent in the generated string, a random position is replaced with one from that class.
  • Entropy — calculated as log₂(charset_size) × length, the theoretical maximum for uniform random.
